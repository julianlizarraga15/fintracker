Daily Valuation Flow

daily at 16:00 UTC
systemd timer 16:00 UTC
docker exec fintracker-backend
run script run_valuations.sh
status wrapper run_valuations_with_status.sh
snapshot job backend.core.daily_snapshot
status files data/job_runs/valuations/

Repo Shape

main folders
backend/ API, source clients, transforms, valuation logic.
frontend/ Static pages served by the frontend container.
scripts/ Manual and scheduled helper commands.
.github/ Build, push, and deploy workflow.
Config .env is local/container env config. This guide does not include secrets or values.

Cheat Sheet

where to look
Question Start Here What It Tells You
What runs daily? scripts/run_valuations.sh Entry point for the scheduled valuation job.
What does the job do? backend/core/daily_snapshot.py Source loading, pricing, FX, valuation, snapshot writes.
Where are API routes? backend/app/routers/ Auth, valuation, job, and price endpoints.
Where are saved results? data/{positions,prices,fx,valuations}/dt=... CSV and best-effort Parquet snapshots.
Where is job health? data/job_runs/valuations/ Run logs, latest status, and history JSON.
How does prod update? .github/workflows/ci-cd.yml Build, push to ECR, deploy through SSM, restart Compose.
